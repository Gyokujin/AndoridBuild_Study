SDK(Software Development Kit) : 소프트웨어를 개발할 때 필요한 도구
NDK(Native Development Kit) : 자신의 플랫폼이 아닌 다른 플랫폼을 통해 프로그래밍을 할 수 있는 도구. 안드로이드에서 c++ 프로그래밍이 필요할 때
JDK(Java Development Kit) : Java API 개발에 필요한 도구

ADB(Android Debug Bridge) : 안드로이드 기기에 연결하여 조작할 수 있도록 돕는다.
그래들 빌드 : 안드로이드의 빌드 설정을 조정할 수 있는 스크립트

그래픽 API : OpenGL, OpenGL ES, Direct X

파일을 저장하는 형식 : 텍스트, 바이너리 형식이 있다.
텍스트 : 파일을 읽을 수 있는 글자로 저장하는 것(인코딩 : UTF8, ANSI)
바이너리 : 이진 코드라고 하며 컴퓨터 프로세서에 사용하는 명령어. 메모리를 그대로 저장하는 것
시리얼라이즈(Serialize) : 객체를 버퍼화하여 저장한다. 일정한 규칙에 의해 데이터를 일련의 바이트로 변형하는 것
버퍼 : 어떤 데이터를 저장하기 위해 만들어진 데이터 배열. 일시적으로 데이터를 저장하는 메모리 공간

윈도우는 초기에 모든 권한을 주고, 폴더접근을 제한하지 않았기 때문에 보안 문제가 컸다(ex: ActiveX)

AppData : 앱에서만 사용 가능한 폴더, 앱에 필요한 파일을 쓰거나 읽는 것을 저장한다. (Prefs의 읽기, 쓰기도 여기에 포함된다)
오픈소스 : 누구나 보고 사용할 수 있게끔 공개한 소스

스크립팅 백엔드(Scripting Backend) : 스크립트 언어로 작성된 코드를 실행 가능한 형태로 변환하는 컴파일러. 주로 Mono와 IL2CPP가 있다.
- Mono : .Net 프레임워크에서 사용하는 오픈소스 백엔드. 쉬운 접근성과 빠른 빌드가 가능하지만 IL2CPP에 비해 성능이 떨어진다.
- IL2CPP : 코드를 c++로 변환하고, 네이티브 코드로 컴파일하는 백엔드
